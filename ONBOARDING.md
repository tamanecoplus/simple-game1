# 新規参加者向けオンボーディングガイド

## 1. このリポジトリの現状

現在の `simple-game1` リポジトリは、Git 管理のみ初期化された**最小構成のスケルトン**です。

- ルート直下には `.gitkeep` とこの `ONBOARDING.md` があります。
- アプリ本体コード（`src/` など）、ビルド設定、テストコード、依存関係定義ファイルはまだ配置されていません。
- 目標ジャンルは、**イースのような高速で手触りのよい見下ろし型アクションRPG**です。ただし、固有名称・キャラクター・世界観・音楽・アートなどは流用せず、オリジナル作品として設計します。

## 2. 作りたいゲームの方向性

このプロジェクトでは、探索・戦闘・成長を短いサイクルで気持ちよく回せるアクションRPGを目指します。

- **視点**: 2D または 2.5D の見下ろし型フィールド
- **操作感**: テンキーまたは WASD による移動と、スペースキー攻撃の反応が軽快であることを最優先
- **戦闘**: 雑魚戦はテンポよく、ボス戦は攻撃予兆・位置取り・反撃の読み合いを重視
- **探索**: 小さなマップをつなぎ、宝箱、イベント、ショートカット、強敵配置で発見を作る
- **成長**: レベル、装備、回復アイテムなどでプレイヤーの強化を実感できる形にする
- **物語**: 最初は短い導入と目的だけに絞り、システムの面白さを先に検証する

## 3. コードベースの一般的な構造（これから作るべき標準形）

現時点で実ファイルはありませんが、見下ろし型アクションRPGとして保守しやすい構成は次のような形です。

- `src/` : ゲーム本体ロジック
  - `core/` : ゲームループ、状態遷移、セーブ/ロード、設定
  - `input/` : テンキー/WASD キーボード入力、将来のゲームパッド対応、入力バッファ
  - `world/` : マップ、エリア遷移、イベントトリガー、衝突判定
  - `entities/` : プレイヤー、敵、NPC、アイテム、弾/エフェクト
  - `combat/` : 攻撃判定、ダメージ計算、無敵時間、状態異常
  - `ai/` : 敵の行動パターン、ボスのフェーズ管理
  - `ui/` : HP、経験値、装備画面、会話、メニュー
  - `data/` : 敵・装備・アイテムの定義データ
- `assets/` : 画像、音声、フォント、マップデータ
- `tests/` : 単体・結合テスト
- `docs/` : 仕様、設計、運用手順
- `README.md` : セットアップ、実行方法、開発フロー

## 4. 参加者がまず知っておくべき重要事項

1. **最初に作るべきものは「縦切りデモ」**
   - 広い世界全体ではなく、1マップ、1種類の敵、1体のボス、1つの回復手段で遊べる最小デモを目指します。
2. **操作感が最優先**
   - 見下ろし型アクションRPGでは、移動速度、攻撃後硬直、ヒットストップ、無敵時間が面白さを大きく左右します。
3. **技術スタックはブラウザ向けを前提にする**
   - 配布先はブラウザとし、初期候補は **TypeScript + Phaser + Vite** にします。URL共有だけで遊べること、Canvas ベースの2D表現に向くこと、軽量に縦切りデモを作りやすいことを重視します。
4. **データ駆動に寄せる**
   - 敵ステータス、装備、アイテムはコードへ直書きせず、JSON/YAML/ScriptableObject などで調整しやすくします。
5. **開発ルールとテスト/CIが未設定**
   - ブランチ運用、コミット規約、lint/test コマンド、最低限の自動チェックを早期に決めます。

## 5. 最初の MVP（最小実用版）案

まずは 5〜10 分遊べる小さなデモを完成させます。

- プレイヤー
  - テンキーまたは WASD による8方向移動
  - 通常攻撃
  - HP と被ダメージ
- 敵
  - 近接雑魚 1種類
  - 遠距離または突進系の雑魚 1種類
  - 簡単なボス 1体（2〜3パターンの攻撃）
- マップ
  - 開始地点
  - 雑魚戦エリア
  - 宝箱または回復地点
  - ボス部屋
- UI
  - HP表示
  - 簡単な会話/メッセージ
- 成長要素
  - 経験値または装備更新のどちらか一方だけを最初に実装

## 6. 操作仕様（初期案）

最初の実装では、キーボードだけで快適に遊べることを優先します。移動はテンキーと WASD の両方を受け付け、どちらでも同じ挙動にします。

### 6.1 移動入力

- **テンキー移動**
  - `8`: 前進/上方向
  - `4`: 左移動
  - `2`: 後退/下方向
  - `6`: 右移動
  - `7` / `9` / `1` / `3`: 斜め移動
- **WASD移動**
  - `W`: 前進/上方向
  - `A`: 左移動
  - `S`: 後退/下方向
  - `D`: 右移動
  - 2キー同時押し（例: `W` + `D`）で斜め移動

### 6.2 戦闘・メニュー入力

初期実装では操作を絞り、攻撃とメニューだけを割り当てます。回避とスキルは MVP には含めません。

- `Space`: 通常攻撃
- `Esc`: メニュー/ポーズ

斜め移動は速度が速くなりすぎないよう、入力ベクトルを正規化します。攻撃は入力バッファを持たせ、多少早めに押しても次の通常攻撃へつながる手触りを目指します。

## 7. 技術スタック（初期選定）

ブラウザでプレイできることを必須条件にし、初期技術スタックは次を推奨します。

- **言語**: TypeScript
- **ゲームエンジン/ライブラリ**: Phaser
- **開発サーバー/ビルド**: Vite
- **テスト**: Vitest（ロジック中心）
- **静的解析/整形**: ESLint + Prettier
- **データ形式**: JSON（敵、装備、アイテム定義）

この選定により、ブラウザ上で素早く縦切りデモを確認でき、将来的には GitHub Pages などの静的ホスティングにも載せやすくなります。

## 8. 次に学習・着手するとよい順序

1. **操作仕様を実装可能な形に落とし込む**（テンキー/WASD移動、スペースキー攻撃、ESCメニュー）
2. **ブラウザ向け技術スタックで雛形を作る**（TypeScript + Phaser + Vite）
3. **プロジェクト骨格を作成**（`src/`, `assets/`, `tests/`, `docs/`, `README.md`）
4. **プレイヤー操作とカメラを実装**（まずは気持ちよく動くこと）
5. **当たり判定とダメージ処理を実装**（攻撃範囲、無敵時間、ノックバック）
6. **敵AIを1種類実装**（索敵、接近、攻撃、クールダウン）
7. **小さなマップとボス戦を作る**（縦切りデモ化）
8. **テスト/CIを導入**（データ検証、ダメージ計算、ステート遷移）
9. **演出を足して手触りを調整**（ヒットストップ、画面揺れ、SE、エフェクト）
